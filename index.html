TTB · COLA Compliance Prototype

AI-powered label review for TTB compliance agents

The TTB reviews approximately 150,000 COLA label applications per year — with 47 agents handling all of them. Each review means manually checking that every element on the physical label matches what was submitted in the application.

This tool uses Claude Vision to read a label image the same way an agent would, then cross-references every required field against the application data in seconds, surfacing mismatches and violations with the specific CFR citation behind each finding.

What it checks
🔍

Vision-based label reading

Claude reads the label image directly — brand name, class/type, ABV, net contents, producer address, country of origin, and the government warning — handling skewed angles, glare, and low-contrast printing.

⚖️

CFR-grounded verification

Each finding is checked against the specific regulation. Same-field-of-vision for spirits (27 CFR 5.63(a)), ABV abbreviation rules for malt (27 CFR 7.65), appellation triggers for wine (27 CFR 4.25). Rule-based reasoning, not guessing.

⚠️

Government warning enforcement

The most common failure mode. The warning must be verbatim, with GOVERNMENT WARNING: in ALL CAPS and bold, the remainder not bold, separate and apart from all other text. The AI checks every condition and names the violation.

📦

Batch processing

Queue multiple label images and run them all in one pass. Results download as a structured CSV — one row per label, four columns per field (Status / Found / Expected / Note) — ready to open directly in Excel.

📤

Structured output

Every result is clean JSON. Export to CSV, download an HTML report, or copy JSON to pipe into Power Automate, a Teams webhook, or any downstream system.

🔒

No data stored

Label images and application data go directly to Anthropic's API for analysis. Nothing is logged or retained by this application. The API key lives only in the server environment.

How it works
Step 1
Upload the label
Agent uploads a photo or scan of the physical label — JPG, PNG, or WEBP. Encoded and sent securely to Claude.
Step 2
Enter application data
Fill in the COLA application fields. Select beverage type to load the correct CFR rule set for that category.
Step 3
Claude reads the label
Claude Vision reads every element on the label image and compares each field against the application data and CFR requirements.
Step 4
Review findings
Per-field results table: Pass, Warn, or Fail — with the exact text found, what was expected, and the specific issue. Agent makes the final call.
Step 5
Export or dispatch
Download CSV or HTML report. Copy JSON. Route automatically to Microsoft Teams or Power Automate.

Regulatory coverage
Distilled Spirits

27 CFR Part 5

  • Same field of vision — brand, ABV, class/type (§5.63(a))
  • Alcohol as % by volume; proof alone not OK (§5.65)
  • Name and address of bottler/distiller (§5.66–5.68)
  • Age statement for whiskies under 4 years (§5.74)
  • Neutral spirits source disclosure (§5.75)
27 CFR Part 5 Subpart A ↗
Malt Beverage

27 CFR Part 7

  • Brand name (§7.64)
  • Class/type — Beer, Ale, Lager, Porter (§7.141)
  • ABV only mandatory if from added flavors (§7.63)
  • “ABV” abbreviation not acceptable (§7.65)
  • Name and address (§7.66–7.68)
  • Allergen/additive declarations (§7.63(b))
27 CFR Part 7 Subpart A ↗
Wine

27 CFR Part 4

  • Brand name and class/type on brand label (§4.33–4.34)
  • “Rosé” alone is not a valid class/type
  • ABV mandatory if >14%; “ABV” not acceptable (§4.36)
  • Appellation of origin triggers (§4.25)
  • Grape variety sourcing ≥75% / 85% (§4.23)
  • Sulfite declaration if ≥10 ppm (§4.32(e))

All Beverages

27 CFR Part 16

  • Mandatory on all beverages ≥0.5% ABV (§16.20)
  • “GOVERNMENT WARNING:” ALL CAPS + bold (§16.22)
  • Remainder must NOT be bold
  • Separate and apart from all other text (§16.21)
  • Min. type size: 1mm / 2mm / 3mm by volume (§16.22)
  • Continuous — cannot be split across labels
